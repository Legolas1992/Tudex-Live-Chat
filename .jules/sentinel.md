## 2026-03-05 - [NoSQL Injection / Object Type Confusion]
**Vulnerability:** The application was vulnerable to NoSQL injection and Object Type Confusion in its authentication logic (`req.headers`, `req.query`, and socket token validation) and ReDoS in regex search parameters. This allowed attackers to bypass authentication by passing objects like `?api_key[$ne]=null` instead of strings.
**Learning:** In the current Express/Mongoose stack, unvalidated query strings and headers can be parsed as objects instead of strings, triggering NoSQL injection when directly queried.
**Prevention:** To prevent Object Type Confusion/NoSQL injection, always explicitly cast unvalidated user request properties (e.g., `req.query`, `req.headers`, `socket.handshake`) to primitives (like `String()`) before evaluating or passing them to the database driver. Always escape input correctly using `replace(/[.*+?^${}()|[\]\\]/g, '\\$&')` when passing dynamically constructed strings to MongoDB's `$regex`.
